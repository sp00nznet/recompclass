# MOS 6502 (NES CPU) Quick Reference

## Overview

The MOS 6502 is an 8-bit processor with a minimal register set and rich addressing modes. The NES uses the Ricoh 2A03 variant, which removes BCD (decimal mode) and integrates an APU for sound generation. The 6502 family also appears in the Commodore 64, Apple II, Atari 2600/7800, and many arcade boards.

- Clock: 1.79 MHz (NTSC NES), 1.66 MHz (PAL NES)
- 8-bit data bus, 16-bit address bus (64 KB addressable)
- Little-endian (16-bit values stored low byte first)

---

## Registers

| Register | Width  | Description                              |
|----------|--------|------------------------------------------|
| A        | 8-bit  | Accumulator                              |
| X        | 8-bit  | Index register X                         |
| Y        | 8-bit  | Index register Y                         |
| SP (S)   | 8-bit  | Stack pointer (stack at $0100-$01FF)     |
| PC       | 16-bit | Program counter                          |
| P        | 8-bit  | Processor status (flags)                 |

The register set is tiny by any standard. Most work flows through the accumulator, with X and Y used for indexing and loop counting.

### Processor Status Register (P)

| Bit | Flag | Name          | Set when...                             |
|-----|------|---------------|-----------------------------------------|
| 7   | N    | Negative      | Bit 7 of result is set                  |
| 6   | V    | Overflow      | Signed arithmetic overflow              |
| 5   | -    | (unused)      | Always 1                                |
| 4   | B    | Break         | Set by BRK, clear on IRQ/NMI push       |
| 3   | D    | Decimal       | BCD mode for ADC/SBC (disabled on 2A03) |
| 2   | I    | IRQ disable   | Masks IRQ (not NMI)                     |
| 1   | Z    | Zero          | Result is zero                          |
| 0   | C    | Carry         | Unsigned carry / borrow (inverted)      |

## Addressing Modes

The 6502 has 13 addressing modes. Every instruction supports only a subset of them.

| Mode                      | Syntax       | Bytes | Effective Address                      |
|---------------------------|-------------|-------|----------------------------------------|
| Implied                   | (none)      | 1     | Operand is implicit (e.g., TAX)        |
| Accumulator               | A           | 1     | Operates on A (e.g., ASL A)           |
| Immediate                 | #val        | 2     | Operand is the literal value           |
| Zero Page                 | zp          | 2     | $00:zp (first 256 bytes)              |
| Zero Page,X               | zp,X        | 2     | ($00:zp + X) & $FF (wraps in ZP)      |
| Zero Page,Y               | zp,Y        | 2     | ($00:zp + Y) & $FF (wraps in ZP)      |
| Absolute                  | addr        | 3     | 16-bit address                         |
| Absolute,X                | addr,X      | 3     | addr + X (page-cross penalty)          |
| Absolute,Y                | addr,Y      | 3     | addr + Y (page-cross penalty)          |
| Indirect                  | (addr)      | 3     | 16-bit pointer at addr (JMP only)      |
| Indexed Indirect          | (zp,X)      | 2     | Pointer at (zp+X) in zero page         |
| Indirect Indexed          | (zp),Y      | 2     | Pointer at zp, then add Y              |
| Relative                  | offset      | 2     | PC + signed 8-bit offset (branches)    |

**Zero page is fast:** Zero page addressing saves a byte and a cycle over absolute addressing. NES games treat zero page as a register file extension.

**Indirect JMP bug:** On NMOS 6502 (including 2A03), `JMP ($xxFF)` reads the low byte from $xxFF and the high byte from $xx00 instead of from $xx00+$0100. This is a hardware bug that games sometimes depend on.

## Key Instructions

| Category    | Instructions                               | Notes                              |
|-------------|--------------------------------------------|------------------------------------|
| Load/Store  | LDA, LDX, LDY, STA, STX, STY              | Only LDA/STA support all modes     |
| Transfer    | TAX, TAY, TXA, TYA, TSX, TXS              | TXS does NOT set flags             |
| Arithmetic  | ADC, SBC, INC, DEC, INX, INY, DEX, DEY    | ADC/SBC always use carry           |
| Logic       | AND, ORA, EOR, BIT                         | BIT sets N/V from memory bits 7/6  |
| Shift       | ASL, LSR, ROL, ROR                         | Operate on A or memory             |
| Compare     | CMP, CPX, CPY                              | Subtract without storing result    |
| Branch      | BEQ, BNE, BCS, BCC, BMI, BPL, BVS, BVC    | 8-bit signed relative offset       |
| Jump        | JMP, JSR, RTS, RTI, BRK                    | JSR pushes PC-1; RTS pulls PC+1   |
| Stack       | PHA, PLA, PHP, PLP                         | Push/pull A or P                   |
| Flag        | SEC, CLC, SED, CLD, SEI, CLI, CLV         | Direct flag manipulation           |
| Misc        | NOP                                        | No operation                       |

### Important Details

- **ADC/SBC always include carry.** To add without carry: `CLC` then `ADC`. To subtract: `SEC` then `SBC`. Forgetting CLC/SEC is a classic 6502 bug.
- **No 16-bit arithmetic.** Multi-byte math is done by chaining ADC/SBC with the carry flag across bytes.
- **JSR/RTS quirk:** JSR pushes the address of the last byte of the JSR instruction (PC-1). RTS pulls this and adds 1. This matters if you're synthesizing calls.

## NES-Specific: Ricoh 2A03

| Feature             | Detail                                     |
|---------------------|--------------------------------------------|
| Decimal mode         | **Removed.** SED/CLD exist but D flag has no effect on ADC/SBC. |
| APU (Audio)          | 2 pulse, 1 triangle, 1 noise, 1 DMC channel, mapped to $4000-$4017 |
| DMA (OAM)           | Write to $4014 triggers 256-byte DMA to PPU OAM (halts CPU 513-514 cycles) |
| Controller I/O       | $4016 (controller 1), $4017 (controller 2) |

## Memory Map (NES)

| Address Range    | Size   | Description                        |
|------------------|--------|------------------------------------|
| $0000 - $00FF    | 256 B  | Zero Page (fast access)            |
| $0100 - $01FF    | 256 B  | Stack                              |
| $0200 - $07FF    | 1.5 KB | RAM                                |
| $0800 - $1FFF    | --     | Mirrors of $0000-$07FF             |
| $2000 - $2007    | 8 B    | PPU registers                      |
| $2008 - $3FFF    | --     | Mirrors of $2000-$2007             |
| $4000 - $4017    | 24 B   | APU and I/O registers              |
| $4018 - $401F    | 8 B    | Test mode registers (normally disabled) |
| $4020 - $5FFF    | --     | Cartridge expansion area           |
| $6000 - $7FFF    | 8 KB   | Cartridge PRG RAM (battery save)   |
| $8000 - $BFFF    | 16 KB  | PRG ROM lower bank                 |
| $C000 - $FFFF    | 16 KB  | PRG ROM upper bank                 |

**Vectors (top of address space):**
- $FFFA-$FFFB: NMI vector
- $FFFC-$FFFD: Reset vector
- $FFFE-$FFFF: IRQ/BRK vector

## Quick Comparison: 6502 vs SM83 (Game Boy)

| Feature          | 6502                    | SM83                        |
|------------------|-------------------------|-----------------------------|
| Width            | 8-bit                   | 8-bit                       |
| Registers        | A, X, Y (3 GPRs)       | A, B, C, D, E, H, L (7)    |
| Register pairs   | None                    | BC, DE, HL                  |
| Flags            | N, V, -, B, D, I, Z, C | Z, N, H, C                  |
| Address bus      | 16-bit                  | 16-bit                      |
| Zero/direct page | $0000-$00FF (fast)      | $FF00-$FFFF (high page)     |
| Stack            | Fixed at $0100-$01FF    | Anywhere (16-bit SP)        |
| Indirect modes   | Rich (indexed, etc.)    | Minimal ((HL), (BC), (DE))  |
| Bit operations   | None                    | CB-prefixed BIT/SET/RES     |
| 16-bit ALU       | No (carry-chain only)   | 16-bit ADD HL,rr / INC rr   |

## Recompiler Pain Points

- **Carry-based arithmetic:** ADC and SBC always include the carry flag. The recompiler must track carry state precisely through multi-byte arithmetic chains.
- **Zero page as register file:** NES games use zero page heavily as extra registers. Mapping hot ZP locations to host registers is a major optimization opportunity.
- **Bank switching (mappers):** NES cartridges use dozens of mapper chips (MMC1, MMC3, etc.) to swap PRG and CHR banks. The recompiler must detect writes to mapper registers and invalidate/redirect affected code.
- **Self-modifying code:** The 6502 has no cache, so self-modifying code works perfectly. Some games patch instruction operands at runtime.
- **Indirect JMP page-crossing bug:** The NMOS 6502 hardware bug on `JMP ($xxFF)` must be emulated exactly -- some games trigger it intentionally.
- **Decimal mode (non-NES targets):** If targeting the original 6502 (not 2A03), BCD arithmetic via the D flag must be handled. BCD ADC/SBC behavior is complex.
- **Cycle-accurate timing:** NES games are often cycle-sensitive due to PPU/CPU synchronization (raster effects, sprite 0 hit). This constrains how aggressively the recompiler can reorder or batch work.
- **Status flag complexity:** BIT instruction sets N and V from memory bits 7 and 6 while setting Z from AND result. This unusual flag behavior requires special handling.
