# 65816 (SNES CPU) Quick Reference

## Overview

The WDC 65C816 is a 16-bit extension of the MOS 6502. Used in the Super Nintendo (Ricoh 5A22 variant), Apple IIGS, and other systems. It maintains full backward compatibility with the 6502 via "emulation mode" while adding 16-bit registers, a 24-bit address bus (16 MB), and many new addressing modes.

- Clock: 1.79 MHz (slow), 2.68 MHz (fast), 3.58 MHz (fastest) -- varies by memory region
- 24-bit address bus, 8-bit data bus

---

## Registers

| Register | Width     | Description                              |
|----------|-----------|------------------------------------------|
| A (C)    | 8 or 16   | Accumulator (C = full 16-bit name)       |
| X        | 8 or 16   | Index register X                         |
| Y        | 8 or 16   | Index register Y                         |
| SP (S)   | 16-bit    | Stack pointer (8-bit in emulation mode)  |
| PC       | 16-bit    | Program counter (within current bank)    |
| P        | 8-bit     | Processor status (flags)                 |
| DBR (B)  | 8-bit     | Data Bank Register (bits 16-23 of data)  |
| PBR (K)  | 8-bit     | Program Bank Register (bits 16-23 of PC) |
| D        | 16-bit    | Direct Page register (relocatable DP)    |

### Processor Status Register (P)

| Bit | Flag | Name             | Notes                           |
|-----|------|------------------|---------------------------------|
| 7   | N    | Negative         | Sign bit of result              |
| 6   | V    | Overflow         | Signed overflow                 |
| 5   | M    | Memory/Acc width | 0=16-bit A, 1=8-bit A          |
| 4   | X    | Index width      | 0=16-bit X/Y, 1=8-bit X/Y     |
| 3   | D    | Decimal mode     | BCD arithmetic                  |
| 2   | I    | IRQ disable      |                                 |
| 1   | Z    | Zero             |                                 |
| 0   | C    | Carry            |                                 |

In **emulation mode** (E=1), bits 5 and 4 revert to the 6502's B and unused bit.

## M/X Flag Register Width

This is the single most important concept for 65816 recompilation.

| M Flag | X Flag | A width | X/Y width |
|--------|--------|---------|-----------|
| 0      | 0      | 16-bit  | 16-bit    |
| 0      | 1      | 16-bit  | 8-bit     |
| 1      | 0      | 8-bit   | 16-bit    |
| 1      | 1      | 8-bit   | 8-bit     |

Changed via `SEP` (set bits) and `REP` (reset bits) instructions:
- `SEP #$30` -- set M and X (8-bit A, 8-bit X/Y)
- `REP #$30` -- reset M and X (16-bit A, 16-bit X/Y)

## Recompiler Pain Points

- **Variable register width (M/X flags):** The same opcode operates on 8-bit or 16-bit data depending on runtime flags. The recompiler must track flag state across branches -- often requires static analysis or worst-case assumptions.
- **24-bit bank addressing:** Code and data can span 256 banks of 64 KB each. Long jumps (JML/JSL) change the PBR; data bank (DBR) is separate. Pointer indirection can cross bank boundaries.
- **Emulation mode:** The 6502-compatible mode changes stack, register, and addressing behavior. Some SNES games switch between modes (rare but possible).
- **Direct Page relocation:** DP can be moved anywhere in Bank 0, making DP-addressed memory a moving target.
- **Block move (MVN/MVP):** These instructions change DBR as a side effect and loop internally. They are interruptible mid-transfer.
- **BCD mode:** Decimal flag affects ADC/SBC results. Rarely used on SNES but must be handled correctly.
- **Stack-relative addressing:** Used heavily for local variables in compiled code; requires accurate SP tracking.
- **Self-modifying code:** The 65816 has no instruction cache, so self-modifying code works and some games use it.
