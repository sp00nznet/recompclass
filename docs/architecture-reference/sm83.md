# SM83 (Game Boy CPU) Quick Reference

## Overview

The SM83 is an 8-bit CPU used in the Game Boy and Game Boy Color. It is often called a "modified Z80" but is more accurately a hybrid between the Intel 8080 and Zilog Z80. It lacks the Z80's IX/IY index registers and most of the Z80 extended instructions, but adds unique bit-manipulation and auto-increment/decrement memory instructions.

- Clock: 4.194304 MHz (GB), 8.388608 MHz (GBC double-speed)
- 8-bit data bus, 16-bit address bus (64 KB addressable)

## Registers

| Register | Width  | Notes                            |
|----------|--------|----------------------------------|
| A        | 8-bit  | Accumulator                      |
| F        | 8-bit  | Flags (not directly accessible)  |
| B, C     | 8-bit  | General purpose / BC pair        |
| D, E     | 8-bit  | General purpose / DE pair        |
| H, L     | 8-bit  | General purpose / HL pair        |
| SP       | 16-bit | Stack pointer                    |
| PC       | 16-bit | Program counter                  |

**16-bit pairs:** AF, BC, DE, HL (high byte listed first)

### Flags Register (F)

| Bit | Flag | Name        | Set when...                         |
|-----|------|-------------|-------------------------------------|
| 7   | Z    | Zero        | Result is zero                      |
| 6   | N    | Subtract    | Last operation was subtraction       |
| 5   | H    | Half-Carry  | Carry from bit 3 to bit 4           |
| 4   | C    | Carry       | Carry from bit 7 / borrow           |
| 3-0 | --   | (unused)    | Always zero                         |

## Addressing Modes

| Mode               | Syntax        | Example          |
|--------------------|---------------|------------------|
| Immediate 8-bit    | n             | `LD A, $42`      |
| Immediate 16-bit   | nn            | `LD BC, $1234`   |
| Register           | r             | `LD A, B`        |
| Register indirect  | (HL)          | `LD A, (HL)`     |
| Register indirect  | (BC), (DE)    | `LD A, (BC)`     |
| Absolute           | (nn)          | `LD A, ($FF44)`  |
| High-page          | ($FF00+n)     | `LDH A, ($44)`   |
| High-page register | ($FF00+C)     | `LDH A, (C)`     |

| HL post-decrement  | (HL-)         | `LD A, (HL-)`    |
| SP-relative        | SP+e          | `LD HL, SP+$02`  |

| Category    | Instructions                          | Notes                           |
|-------------|---------------------------------------|---------------------------------|
| Load        | LD, LDH, PUSH, POP                   | LDH is high-page shortcut      |
| Arithmetic  | ADD, ADC, SUB, SBC, INC, DEC         | 8-bit and 16-bit variants      |
| Logic       | AND, OR, XOR, CP                      | CP = compare (SUB without store)|
| Rotate/Shift| RLCA, RRCA, RLA, RRA, RLC, RRC, RL, RR, SLA, SRA, SRL, SWAP | CB-prefixed except xCA/xRA |
| Bit ops     | BIT, SET, RES                         | CB-prefixed, test/set/reset bit |
| Control     | JP, JR, CALL, RET, RETI, RST         | Conditional: Z, NZ, C, NC      |
| Misc        | NOP, HALT, STOP, DI, EI, DAA, CPL, SCF, CCF | DAA = decimal adjust       |

## Memory Map

| Address Range     | Size   | Description               |
|-------------------|--------|---------------------------|
| $0000 - $3FFF     | 16 KB  | ROM Bank 0 (fixed)        |
| $4000 - $7FFF     | 16 KB  | Switchable ROM Bank        |
| $8000 - $9FFF     | 8 KB   | Video RAM (VRAM)           |
| $A000 - $BFFF     | 8 KB   | External (Cartridge) RAM   |
| $C000 - $DFFF     | 8 KB   | Work RAM (WRAM)            |
| $E000 - $FDFF     | --     | Echo RAM (mirror of WRAM)  |
| $FE00 - $FE9F     | 160 B  | OAM (Sprite attributes)   |
| $FEA0 - $FEFF     | --     | Unusable                   |
| $FF00 - $FF7F     | 128 B  | I/O Registers              |
| $FF80 - $FFFE     | 127 B  | High RAM (HRAM)            |
| $FFFF             | 1 B    | Interrupt Enable register  |

## Recompiler Pain Points

- **Bank switching:** ROM and RAM banks are swapped at runtime by writing to MBC registers mapped into ROM space ($0000-$7FFF). The recompiler must detect these writes and redirect execution to the correct bank.
- **HALT and STOP:** HALT waits for an interrupt; STOP halts the CPU and LCD until a button is pressed. Both require careful idle-loop handling.
- **DAA instruction:** Decimal Adjust Accumulator is notoriously tricky. It adjusts the result of a binary add/sub into BCD using both H and N flags. Many recompilers get this wrong.
- **Bit operations (CB prefix):** 256 CB-prefixed opcodes for bit test/set/reset and shifts. Large decode surface area.
- **Echo RAM:** Writes to $E000-$FDFF mirror to $C000-$DFFF. Some games rely on this.
- **OAM DMA:** Transfers 160 bytes to OAM; CPU can only access HRAM during DMA.
- **Interrupts and RETI:** Interrupt dispatches must be modeled accurately since RETI re-enables interrupts atomically.
